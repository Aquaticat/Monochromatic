  Navigation Architecture Recommendation

  Primary recommendation: Abandon the cycle button, use drawer views

  Here's why the cycle button approach has significant UX problems:

  1. Discoverability: Users won't know what states exist or how to reach them
  2. Context loss: Cycling forces users through unwanted views to reach their target
  3. No visual preview: Unlike a drawer/list, users can't see counts or status at a glance
  4. Scales poorly: Works for 2 states, breaks if you add a 3rd view later

  Specific Answers to Your Questions

  "Should I use a cycle button instead?"

  No, for these reasons:

  1. The current topNav label ("Unsorted") is clearer - it tells users exactly where they are
  2. A cycle button hides information - users can't see counts for other views
  3. Drawer navigation is expected - users know to look there for navigation options
  4. You'll want more than 2 views eventually - "AI Needs Review", "Recent", "Favorites" are natural additions

  "How should drawer relate to topNav?"

  Proposed relationship:

  | Element                | Purpose                | Content                                            |
  |------------------------|------------------------|----------------------------------------------------|
  | Top Nav                | Context + Actions      | Current view name, search, add task, drawer toggle |
  | Drawer                 | Navigation + Discovery | All views, filters, tag browsing, settings         |
  | Filter Bar             | Active state           | Shows applied tag filters as dismissible chips     |
  | Floating Action Button | Primary action         | Quick add task (optional, can use + in topNav)     |

  The topNav shows where you are and what you can do, while the drawer shows where you can go and what exists.

  Summary of Key Design Decisions

  1. Segmented control in top nav: Fast binary toggle, always visible counts
  2. Dedicated AI conversation (ğŸ’¬): Prominent placement for core feature
  3. Global search: Dedicated view, searches all tasks, shows source view
  4. No filtering in Sorted: Preserves AI prioritization integrity
  5. Context input separated from AI chat: Structured vs conversational data
  6. Drawer for utilities: Settings, context, tag browsing (not navigation)
  7. Celebration on Unsorted = 0: Positive reinforcement for task processing
